# AI 叙事编辑与 Zine 排版能力设计

- 日期：2026-07-11
- 状态：已确认设计，待实施计划
- 范围：Web 与 Desktop 叙事编辑器；Desktop Zine 编辑器
- 核心体验：AI 直接代替用户编辑；一轮任务只产生一个撤销步骤

## 1. 目标与关键决策

本功能为 MO Gallery 增加三类 AI 编辑能力：

1. 叙事编辑器内容优化；
2. 叙事编辑器结构和排版优化；
3. Desktop Zine 当前跨页的内容、素材、裁切和版式优化。

已确认的产品决策：

| 维度 | 决策 |
|---|---|
| 应用方式 | AI 结果直接写入编辑区域，不显示确认弹窗；Diff 统一显示在 AI 侧栏对话中 |
| 历史粒度 | 一轮 AI 任务等于一个原子编辑操作；一次撤销恢复整轮修改 |
| 执行期间 | 完全锁定目标编辑器，Zine 也禁止切换跨页 |
| 交互入口 | 固定右侧 AI 侧栏，替换叙事编辑器现有浮动助手；Zine 新增同类侧栏 |
| 触发方式 | 对话任务和一键页面/跨页体检 |
| 叙事范围 | 读取当前页面完整内容和结构；不提供选区优化入口 |
| Zine 范围 | 默认修改当前跨页；可参考全项目和相邻跨页，但不得修改其他跨页 |
| 视觉输入 | 结构数据与页面截图结合；模型不支持视觉时降级为结构分析 |
| 删除 | 默认禁止；只有用户本轮明确要求删除时才开放最小范围删除能力 |
| 新增 | 可新增文本框和图片框；Zine 可使用当前项目素材库中的既有素材 |
| 会话 | 叙事和 Zine 均支持多会话；会话属于文档或项目，不属于具体跨页 |

## 2. 总体架构

采用“结构化操作协议 + AI 沙箱执行 + 原子提交”的混合方案，兼顾类似 Notion/Figma 的代操作体验与本地编辑器安全性。

```text
用户任务
  -> 锁定编辑器
  -> 采集结构化快照和视觉上下文
  -> AI 在内存副本中规划并执行
  -> 本地确定性规则校验
  -> 生成最终结构化操作批次
  -> 一次性提交正式编辑器
  -> 写入一个撤销记录
  -> 解锁编辑器
```

### 2.1 共享 AI 核心

扩展 `packages/ai-agent`，保持其为 Web、Desktop、TipTap 与 Zine 的共享 AI 编排层：

```text
packages/ai-agent
├── 通用任务、事件、会话和模型能力协议
├── 叙事文档快照与操作协议
├── Zine 项目快照与操作协议
├── 内容、版式和体检提示词
├── 模型能力检测与降级
└── 沙箱任务执行器
```

约束：

- 提示词只存在于共享 TypeScript 包，不复制到 Web、Desktop 或 Go；
- Web 通过 Hono 代理，由服务端注入模型密钥；
- Desktop 通过本地 Go 代理注入密钥；
- Go 不承载提示词或排版决策；
- `packages/ai-agent/src/domain/**` 不依赖 Vercel AI SDK 类型；
- AI 不直接调用 TipTap command 或 Zustand setter。

### 2.2 宿主事务适配器

共享 AI 层只输出中立操作，由宿主适配器负责模拟、校验和提交：

```ts
interface AiDocumentHost<Snapshot, Operation> {
  captureSnapshot(): Promise<Snapshot>;
  simulate(snapshot: Snapshot, operations: Operation[]): SimulationResult;
  commit(operations: Operation[]): CommitResult;
  lock(): void;
  unlock(): void;
}
```

- TipTap 适配器将整轮操作组合到一个 transaction 和一个历史边界中；
- Zine 适配器先修改克隆项目，通过校验后一次更新 Store、一次压入历史并触发一次保存；
- 任一操作失败时整批不提交；
- 中间尝试不渲染到正式编辑区域。

## 3. AI 侧栏与会话交互

### 3.1 侧栏形态

叙事编辑器以固定右侧侧栏替换现有浮动助手；Desktop Zine 使用一致的视觉语言和任务状态。

侧栏包含：

- 新建、切换、重命名、清空和删除会话；
- 模型选择；
- 对话和任务记录；
- Story、Blog 和 Zine 共用的对话内 Diff 卡片；
- 一键“分析当前页面”或“分析当前跨页”；
- 任务进度、停止按钮、修改摘要、规则警告和降级提示；
- 文本输入和发送按钮。

工具栏增加 AI 图标：

- 点击打开或收起侧栏；
- 运行时显示任务状态；
- 收起侧栏不停止任务，编辑器继续保持锁定；
- 宽屏采用并排侧栏，窄窗口采用覆盖侧栏；
- Web/Desktop 叙事编辑器共用侧栏实现；Zine 复用交互组件但使用 Zine 专属任务适配器。

### 3.2 对话内统一 Diff

Story、Blog 和 Zine 的 AI 修改统一使用同一个对话内变更集协议和 Diff 卡片。Diff 不弹窗、不要求确认，也不阻塞自动应用。编辑区域只显示本轮修改完成后的最终效果。

每轮成功任务在对应助手消息中展示：

- 修改目标：Story、Blog 页面或 Zine 跨页；
- 内容变化：文本新增、删除和替换；
- 结构变化：节点或槽位新增、删除、移动和重排；
- 属性变化：标题层级、Marks、对齐、字体、颜色、尺寸和行高；
- Zine 视觉变化：素材替换、位置尺寸、旋转、层级和裁切参数；
- 修改摘要、规则警告、模型降级信息和应用状态。

统一 Diff 使用结构化条目表达差异，而不是为三种编辑器分别实现不兼容的结果 UI：

```ts
interface AiChangeSet {
  taskId: string;
  targetLabel: string;
  entries: AiChangeEntry[];
  warnings: AiTaskWarning[];
  state: 'applied' | 'undone' | 'redone';
}

interface AiChangeEntry {
  operation: string;
  targetId: string;
  targetLabel: string;
  category: 'content' | 'structure' | 'style' | 'asset' | 'layout';
  before?: unknown;
  after?: unknown;
}
```

渲染层根据条目类别选择紧凑表现：文本使用行内增删高亮，属性和几何变化使用前后值，结构和素材变化使用带目标名称的变更描述。三种编辑器共享卡片框架、状态和交互，不在编辑画布内叠加 Diff 标记或修改前预览。

Diff 卡片随会话记录持久化，保存结构化的必要前后值，但不保存页面截图、跨页预览或素材二进制。卡片提供“撤销本轮修改”；只有该任务仍对应当前可撤销历史项时按钮才可用，避免绕过编辑器历史顺序。撤销后卡片显示“已撤销”，重做后显示“已重做”。

### 3.3 会话作用域

建议作用域：

```text
story:<storyId>
blog:<blogId>
zine:<projectId>
```

规则：

- 会话属于当前 Story、Blog 或 Zine 项目；
- Zine 会话不绑定跨页，切换跨页不会自动创建或切换会话；
- 新建会话只清空新的聊天上下文，不改变当前编辑位置或项目内容；
- 每轮执行前重新采集真实文档，聊天历史不能代替实时状态；
- Zine 任务消息记录目标 `spreadId` 和跨页序号；
- 多个会话类似同一代码项目中的多个 Codex 对话：历史独立，但均读取项目当前状态。

### 3.4 任务入口

#### 对话任务

用户可描述整体或局部目标，例如：

- “优化这一页的叙事节奏并统一小标题。”
- “把第二个文本框写得更简洁。”
- “让当前跨页与前一跨页更连贯。”
- “从项目素材库选择更适合右页的照片。”

系统不提供选区优化模式。AI 通过稳定节点/槽位 ID、类型、文本摘要和位置描述识别目标。若自然语言目标无法唯一匹配，AI 先追问，此时不锁定编辑器，也不修改内容。

#### 页面体检

体检不是只输出报告，而是：

1. 执行确定性规则检查；
2. 执行结构与视觉分析；
3. 自动修复可安全处理的问题；
4. 将全部修复作为一个原子任务提交；
5. 报告不能可靠自动修复的警告。

例如低分辨率原图只能告警，不能伪造修复结果。

### 3.5 任务生命周期

统一状态：

```text
准备上下文
-> 分析页面
-> 规划修改
-> 在副本中执行
-> 校验结果
-> 应用修改
-> 已完成
```

执行期间：

- TipTap 设为不可编辑；
- Zine 禁用 Moveable、文本编辑、拖放、模板、页条和跨页切换；
- 禁止撤销、重做、保存、再次发送任务、切换会话或模型；
- 侧栏仅允许查看状态和停止任务；
- 离开页面或关闭窗口时显示任务进行中的确认。

停止、失败或校验不通过时：

- 取消模型请求和后续步骤；
- 丢弃沙箱中的所有变化；
- 不写入编辑器历史；
- 解锁编辑器；
- 将任务标记为已停止或失败。

成功后编辑区域直接展示最终结果，对话区域显示统一 Diff 卡片、摘要和“本轮修改可一次撤销”的提示。

## 4. 结构化快照与视觉上下文

### 4.1 叙事页面快照

每轮任务采集：

- 完整 TipTap JSON；
- 稳定节点 ID、类型、层级和父子关系；
- 文本、Marks、对齐、链接和节点属性；
- 图片、图片组、表格、引用、链接卡片和媒体结构；
- 编辑区域有效宽度；
- 当前文档修订版本；
- 编辑区域压缩截图。

截图规则：

- 只截取编辑内容，不包含后台导航、AI 侧栏和敏感控件；
- 超长页面按纵向分段，并提供节点与截图分段映射；
- 按模型限制压缩；
- 截图只用于理解视觉结果，正式操作必须指向结构化节点 ID。

### 4.2 Zine 快照

每轮任务采集：

1. **全项目结构摘要**：跨页顺序、角色、模板、槽位、文本层级、素材分配、页面和装订参数、色彩与密度摘要；
2. **当前跨页**：完整结构、较高质量压缩预览、槽位和素材元数据、文字与排版属性；
3. **相邻跨页**：前后跨页缩略图、结构、颜色和密度摘要；
4. **项目素材候选**：从已导入素材中按方向、尺寸、DPI、已用状态和视觉相关性筛选的有限缩略图；
5. **修订版本**：当前跨页及相关项目结构的规范化 revision。

当前跨页是唯一写目标；其他跨页只提供关联设计参考。操作协议和运行时校验都必须拒绝其他 `spreadId`。

## 5. 结构化操作与能力边界

### 5.1 通用操作

- `replace_text`：替换文字；
- `set_node_attrs`：修改节点、文本框或槽位属性；
- `move_node`：移动或重排节点；
- `insert_node`：新增内容节点或槽位；
- `delete_node`：按授权删除；
- `apply_layout_template`：应用既有版式模板；
- Zine 专属素材分配、图片裁切、旋转和层级操作。

操作必须携带稳定目标 ID，并通过严格 Schema 校验。首版不允许模型输出任意 CSS 或直接返回整份替换文档。

### 5.2 叙事编辑能力

内容操作：

- 改写、扩写、缩写、续写、总结；
- 优化语气、节奏、衔接和表达；
- 修改标题、段落、列表、引用和图片说明；
- 拆分或合并段落；
- 新增标题、段落、引用和分隔元素。

排版操作：

- 调整标题层级、对齐和文本 Marks；
- 调整已有段落、标题、图片和图片组属性；
- 优化图片排列、图文顺序和内容分组；
- 使用现有编辑器能力处理分隔线、引用和首字下沉；
- 在当前 TipTap Schema 范围内优化表格、链接卡片和媒体节点。

不自动保存或发布 Story/Blog，不修改页面外壳、主题或其他草稿。

### 5.3 Zine 编辑能力

图片和槽位：

- 移动、缩放、旋转和调整层级；
- 修改图片缩放、偏移、旋转和裁切；
- 新增空图片框；
- 使用当前项目素材库中的既有素材；
- 重新分配当前跨页已有素材；
- 应用现有模板或重新组织当前跨页槽位。

文本和排版：

- 新增标题、正文、引文和说明文本框；
- 改写当前跨页文字；
- 修改字号、行高、字体族、颜色、对齐、位置、尺寸、旋转和层级；
- 参考相邻跨页统一风格、层级和节奏。

AI 不得访问系统文件、外部图库或未导入项目的图片，不得修改或删除素材原文件，也不自动生成或导入图片。

### 5.4 删除授权

默认不向模型开放删除工具。只有本轮用户指令明确包含删除意图时才开放，并遵循：

- 只能删除与指令明确对应的节点或槽位；
- 不能因版面放不下而自行删除；
- 不能删除其他跨页内容；
- 不能删除 Zine 项目素材；
- 删除和本轮其他修改仍组成一个撤销步骤。

## 6. 沙箱、规则校验与原子历史

### 6.1 修订版本

叙事 revision 基于规范化 TipTap 结构；Zine revision 基于稳定序列化后的项目设置、跨页顺序、槽位几何、内容、样式、素材引用和相关素材尺寸/DPI。

对象 URL、更新时间等易变字段不得进入 Zine revision。提交时必须验证 `baseRevision`，防止将结果应用到不同文档状态。

### 6.2 叙事校验

- 目标节点存在且类型支持对应操作；
- 操作不超出当前文档；
- 插入位置和父节点合法；
- 结果满足 TipTap Schema；
- 链接、表格和图片组结构保持合法；
- 未授权删除被拒绝；
- 完成后的文档可被编辑器正常解析。

### 6.3 Zine 校验

- 写操作只指向任务开始时的当前跨页；
- 槽位、素材和模板 ID 存在；
- 数值有限且尺寸有效；
- 页面侧、边界、出血和安全边距合法；
- 文本不溢出；
- 重叠符合允许的层级关系；
- 图片裁切不产生空白；
- 计算并报告有效 DPI；
- 页码区域不被重要内容遮挡；
- 新素材只来自当前项目素材库；
- 未授权删除被拒绝。

### 6.4 校验结果

- **错误**：阻止整批提交；
- **可自动修正错误**：在沙箱内调整后重新校验；
- **警告**：允许提交并在侧栏展示；
- **信息**：用于生成修改摘要。

自动修正设置最大循环次数。超过限制仍不合法时，任务失败且不提交部分结果。

### 6.5 Undo/Redo

- TipTap 用一个 transaction 提交整轮操作；
- Zine 新增批量原子操作接口，在克隆项目上模拟后一次更新 Store；
- 一轮任务只压入一个历史记录；
- 一次撤销恢复该轮全部文本、样式、素材、几何和裁切变化；
- 一次重做恢复完整 AI 结果；
- 撤销或重做不删除会话记录，对话内 Diff 卡片显示已应用、已撤销或已重做状态；
- Diff 卡片中的撤销操作必须调用编辑器原生历史能力，且仅能撤销当前历史栈顶部对应的 AI 任务。

## 7. 页面体检

### 7.1 叙事页面

检查并在可靠时修复：

- 标题层级不连续；
- 段落过长、过密或重复；
- 叙事衔接问题；
- 图文顺序和说明缺失；
- 对齐或样式不一致；
- 图片组结构不协调；
- 页面节奏、留白、异常空节点或无效链接。

### 7.2 Zine 当前跨页

检查并在可靠时修复：

- 图片 DPI、出血和安全边距；
- 文本溢出、槽位越界和异常重叠；
- 裁切对主体的影响；
- 图文层级和视觉重心；
- 与相邻跨页的节奏和风格连续性；
- 素材重复、风格断裂；
- 标题、正文和说明样式一致性。

无法可靠修复的问题只报告，不为追求全绿状态而破坏内容。

## 8. 模型能力与降级

任务启动前检查图片输入、工具调用/结构化输出和上下文能力。

### 8.1 完整模式

支持视觉和结构化操作时，执行完整视觉与结构优化。

### 8.2 结构降级模式

模型不支持视觉时：

- 不发送截图和素材缩略图；
- 继续使用结构、几何、尺寸、DPI 和颜色摘要；
- 不执行依赖主体识别或照片语义的操作；
- 在侧栏明确提示结构分析降级。

### 8.3 受限建议模式

模型不能可靠输出工具调用或严格结构化 JSON 时：

- 仅输出文字建议；
- 不写入编辑器；
- 不产生撤销记录；
- 提示切换到支持结构化操作的模型。

不得将不可校验的自然语言强行解析为编辑命令。

### 8.4 上下文预算

超限时依次：

1. 降低相邻跨页缩略图分辨率；
2. 减少素材候选；
3. 压缩远端跨页摘要；
4. 减少超长叙事页面视觉分段。

当前页面或当前跨页的完整可编辑结构必须保留。仍超限时任务不启动并给出可操作提示。

## 9. 数据持久化、安全与隐私

### 9.1 任务 metadata

普通消息沿用现有会话存储。任务结果保存轻量 metadata：

```ts
interface EditorAiTaskMetadata {
  taskId: string;
  capability: 'narrative' | 'zine';
  taskType: 'instruction' | 'page_audit';
  target: { documentId: string; spreadId?: string };
  status: 'completed' | 'failed' | 'stopped';
  model: string;
  visualMode: 'vision' | 'structure_only';
  summary: string[];
  warningCodes: string[];
  operationSummary: Array<{ type: string; targetIds: string[] }>;
  changeSet: AiChangeSet;
  baseRevision: string;
  resultRevision?: string;
  durationMs: number;
}
```

截图、素材缩略图、完整快照、沙箱副本和模型原始操作输出仅在任务期间驻留内存。

### 9.2 内容边界

叙事任务只发送当前 Story/Blog、必要会话历史和编辑区域截图。Zine 只发送当前项目摘要、当前跨页、相邻跨页和筛选后的项目素材缩略图。

不得发送其他草稿、其他 Zine 项目、系统文件、未导入素材、后台导航或无关控件截图。日志不得记录密钥、原始截图或完整模型请求体。

### 9.3 Web 会话所有权

Web 会话需关联当前用户，所有列表、读取、更新、删除和生成接口校验所有者，避免仅凭 `scopeId` 访问其他管理员的会话。

旧数据迁移必须采用保守策略：无法可靠确定所有者的记录不进入普通用户会话列表，待管理员显式处理。Desktop 为本地单用户数据库，不增加远端所有权模型。

## 10. 取消、错误与恢复

统一 `AbortController` 贯穿截图、上下文准备、模型请求、工具循环、模拟和校验。最终同步提交是不可中断的短事务；提交完成后用户可撤销。

错误分类：

- 模型配置不可用或能力不足；
- 网络、超时或上游错误；
- 截图或素材读取失败；
- 结构化输出无效；
- 目标不存在或删除未授权；
- 沙箱模拟失败；
- Schema、几何、文本或印刷校验失败；
- 本地保存失败。

模型、模拟或校验失败时不改变正式文档。Zine 编辑已提交但 IndexedDB 保存失败时保留本地编辑和 Undo 记录，将项目标记为未保存并允许重试，不自动回滚用户眼前的结果。

## 14. 验收标准

1. 叙事和 Zine 编辑器均可通过工具栏图标打开固定 AI 侧栏。
2. 两类编辑器支持多会话，但会话不绑定具体跨页。
3. AI 自动识别当前页面或跨页，不提供选区优化模式。
4. Zine 可参考全项目和相邻跨页，但只能修改当前跨页。
5. 支持视觉的模型使用结构和截图；不支持时明确降级。
6. AI 运行期间完全锁定编辑器，并支持停止。
7. 成功结果直接写入编辑区，不显示确认弹窗；Story、Blog 和 Zine 的 Diff 统一显示在对话区域。
8. 一轮任务无论修改多少处，只产生一个撤销步骤。
9. 一次撤销完整恢复任务前状态，一次重做恢复任务结果。
10. 失败、停止或校验不通过时不产生部分修改。
11. 默认禁止删除；用户明确要求时仅允许目标明确的删除。
12. Zine 可新增槽位并使用项目素材库，但不能引入外部素材。
13. 页面体检自动修复可靠问题并报告其余警告。
14. Web 与 Desktop 共用提示词、协议和核心逻辑。
15. Zine AI 仅在 Desktop 提供，现有屏幕预览和 PDF 导出保持一致。

## 15. 非目标

- Web 端 Zine 编辑器或 Zine 云同步；
- AI 自动发布 Story/Blog；
- AI 自动导入、生成或删除 Zine 项目素材；
- 未经明确授权的内容删除；
- 任意 CSS 或绕过编辑器 Schema 的修改；
- 在编辑区域中显示 Diff 标记、修改前预览或等待确认的覆盖层；
- 将模型中间步骤逐个写入正式编辑器；
- 首版实现跨任务的自动冲突合并。
